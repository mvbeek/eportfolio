Initial Post: Network and Software Defense Technologies, Fostering Resilient Security Systems

My initial post hinges upon two security technologies which mitigate vulnerabilities in a software database and avert the threat of unauthorised access over a network.

Cryptography can safeguard against a targeted cyber-attack on a network with the adoption of the durable Wi-Fi Protected Access (WPA2) encryption system, which employs the Advanced Encryption Standard (AES) block cipher in counter mode with CBC-MAC (Anderson, 2008).

The Advanced Encryption Standard, consisting of a 128-bit block level encryption surpasses the 64-bit block Data Encryption Standard due to its higher block size and key length, making it considerably difficult for attackers to deduce the key Brooks et al. (2018).

As the contemporary world economy grows further dependent on information technology, businesses operate vital databases which power critical sectors such as e-commerce, inventory, and utilities with the adoption of web servers that directly forward transactions to each respective database. These databases hold data that is imperative to our livelihood such as bank accounts, vehicle registration details, and employment records, which can cause indelible damage in the hands of a cyber-criminal.

Liabilities in a software database can be circumvented with the use of authentication as an access control. Brooks et al. (2018) defines authentication as the process of ascertaining that the user of a database for instance, is who they say they are. (Anderson, 2008) maintains that more efficient and robust authentication techniques adopt the use of multi-factor authentication. This involves a former requirement based on user knowledge such as a password, while the latter condition is an object in the user’s possession, such as a USB key dongle or a RFID key fob.

Consequently, if both authentication factors are challenging to predict, the scenario of a brute-force attack being successful is extremely improbable, relative to the use of single factor authentication.  

References

Brooks, C., Grow, C., Craig, P. & Short, D. (2018) Cybersecurity Essentials. 1st ed. Indianapolis, Indiana: Sybex.

Anderson, R. (2008) Security Engineering: A Guide to Building Dependable Distributed Systems. 2nd ed. Indianapolis: Wiley.

Haroun, great post regarding WPA2 as well as multifactor authentication. Of particular interest is the section where you mention the encryption used in WPA2 and how it safeguards against a cyber-attack. While WPA2 is certainly a vast improvement on its predecessors WEP and WPA. Interestingly WPA2 is vulnerable to a Pixie Dust attack utilising Airgeddon which can crack networks in seconds (Kody, 2018).

As you know WPA2 has different versions such as WPA2-PSK, which is less secure than AES, this protocol is mainly used in small offices, airports and home networks. This wireless standard is widely utilised as it has compatibility with a wide range of devices. However, a brute-force dictionary attack could be performed on this type of wireless network which would allow an attacker to be able to decrypt the traffic, obtain the pre-shared key then capture the handshake when a user joins the network (Goel, 2021).

In today’s wireless filled world it is imperative to safeguard networks in order for the right person to gain access to the right information. To gain an advantage and to keep up with the latest security technologies one should look into new advancements, would you consider WPA3? WPA3-SAE (simultaneous authentication of equals) to replace WPA2-PSK, WPA3-SAE utilises 128-bit encryption key and forward secrecy protocol to protect against dictionary attacks. WPA3-Enterprise to replace WPA2-Enterprise, “which uses a 192-bit encryption key and a 48-bit initialization vector as requested by sensitive organizations” (Goel, 2021).

Initial Post: utilizing IDS and IPS

Network, internet of things devices and wireless network became ubiquitous, and the major concern is how to handle the security issues of these devices and protect them again cyberattacks. Protecting against these attacks is a challenging task, as a result of the heterogeneity of the devices and protocols. Utilizing security technologies would play a vital role providing a real time attack detection and prevention (Hattarki, Houji, Dhage, 2021).

Detecting unwanted traffic on a network or a device can be defined as intrusion detection. IDS is an abbreviation of intrusion detection system, which is a software or a hardware that is installed to monitor the traffic in the network to find and detect unwanted traffic, events and activities, moreover, it can store event logs to be reviewed later. Notifying administrators of the most critical events that are detected, recording information related to the events and producing reports are the main functions performed by the IDS.

IDS technologies have many different types, it depend on the event they are monitoring, and some of these types are: Network Based IDS, host Based IDS, Wireless IDS and Network Behavior Anomaly Detection.

On the other side, intrusion prevention systems (IPS) are the technologies that are used to perform both, detecting threats and intrusion into the system or the network, and managing the responsive actions to these detected threats. IPS monitor malicious traffic at real time generate alerts regarding these traffic as well as it can drop it or block it from the network. If an attack is active, IPS would be the countermeasure to stop that attack. The main functionalities performed by IPS include detection and prevention against malicious activities, stopping the attack, changing attack’s content and altering the security environment.

References

Chakraborty, N. (2013) INTRUSION DETECTION SYSTEM AND INTRUSION PREVENTION SYSTEM: A COMPARATIVE STUDY. International Journal of Computing and Business Research (IJCBR) 4(2): 2-4. Available from: http://researchmanuscripts.com/May2013/1.pdf [date of access 12 June 2021].

Hattarki, R. Houji, S. and Dhage, M. (2021) ‘Real Time Intrusion Detection System for IoT Networks’, Pune, India. Apr 02-04, 2021. Pune, India: Sinhgad College of Engineering.


PEER RESPONSE

Mohamad, great post about utilizing an Intrusion Detection System (IDS) and an Intrusion Prevention System (IPS). Of particular interest is the way you described the role IDS and IPS play in an organization, how they monitor the network as well as detect and block anomalies. Perhaps, what may be useful is to describe the challenges that IDS and IPS cause and how these affect the environment in which they are used.

One of the most frequent problems with IPS is “the detection of false positives or false negatives” (Urrutai, Ierace, Bassett, 2005). When a false positive occurs, the system blocks the activity on the network, assumes the activity is malevolent and causes denial of service to a valid user. Alternately a false negative on the system allows the malicious event to pass by unnoticed. IDS also have their share of problems, mainly caused by the systems producing an abundant number of alerts. One IDS user for example disclosed that he had over 1.8m million alerts monthly (Urrutai, Ierace, Bassett, 2005). As one can imagine this would be an almost impossible amount of work to sift through. Figure 1 (Aljanabi, Ismail, Ali, 2021) shows an interesting diagram highlighting the challenges of an IDS. These systems are also vulnerable; attackers can fool an IDS into thinking an attack is genuine traffic and pass by unnoticed. The techniques used for this type of attack are application hijacking, fragmentation, denial of service and obfuscation (Carlo, 2021).

It would seem both systems require a good amount of finetuning to acquire the desired results. However, considering both systems and even taking into consideration their weaknesses, the consensus seems to be that the benefits out way the long-term advantages as well as the overall performance (Urrutai, Ierace, Bassett, 2005).

Initial Post: Next-Generation Firewalls

A fundamental shift in application and threat landscape, user behavior, and network infrastructure (Media.paloaltonetworks.com. 2021) is one reason we need the next-generation firewall as part of our infrastructure portfolio. 

The next-generation firewall addresses some of the essential key features compared to traditional firewalls, and that is application awareness, integrated Intrusion Prevention Systems, identity awareness which facilitates user and group control, and the ability to use external intelligence sources for different purposes (Clark, C., 2018) 

In an ever-changing application landscape, particular systems and applications can be built to use less traditional network ports, which can be challenging to maintain a security policy. The next-generation firewall allows you to identify applications regardless of the ports used and decide on the security policy based on that (Media.paloaltonetworks.com. 2021). It also facilitates the way users are being allowed or denied access to particular resources when those users are mobile (or connect to different parts of the network that have different subnets). These types of firewalls allow you to grand or deny those accesses based on users and not IP addresses assigned to machines. (Media.paloaltonetworks.com. 2021). 

The most important reason for choosing a next-generation firewall would be that it contains a multitude of systems in addition to a traditional firewall (that blocks IPs and ports) which a company can use for deep packet inspections, IPS, application control without having to purchase or integrate into their separate infrastructure appliances. At the same time, it allows users to keep their mobility and access privileges. (Cloudflare.com. 2021)

Initial Post: Work from Home and Security issues threatening your Company’s IP

As the Covid19 locked downs started, businesses closed doors to their employees and for some times, people lived in somewhat doubts about their financial future and what would happen to their jobs, etc. But thanks to today’s technology such as Microsoft Teams, Zoom, Webex and applications that allow real-time collaboration enabling employees to continue to be productive. With all these technological facilities, almost all businesses were back on but this time in a different shape; Closing all downtown Skyscrapers and working from home or working remotely (Tim Bajarin, 2021).

Working from home came with its challenges, of course. A proper home office, sufficient Internet bandwidth, proper Local Area Network or home Network setup, somewhat quite or free from disruption of children were some of the examples.

One other very interesting challenge that I would like to write about today, is the Security of a home network. There are multiple vulnerable points to threaten your company’s Intellectual Property (IP) working from home when it comes to Cyber Security.

Below are some of the examples that could result disastrous to your company’s IP:

· A personally owned home computer as most home computers are not properly protected by an Antivirus or Firewall. Most home computers do not have proper security patches applied to vulnerable applications, Web Browsers and plugins on time.

· A smart device such as a Mobile Phone, a Tablet specially if used by younger children who would keep tapping on advertisements and possible adware, malware links.

· A home’s Internet Modem or managed/unmanaged Network Switches, the firmware could be easily manipulated or bugged as could be used to sniff all incoming and outgoing network traffic.

· Home’s Wireless device or overall Wireless network security, a weaker Wireless network key and/or out of date security protocols such as Wired Equivalent Privacy.

· Unauthorized physical access of your work computer.

All above examples could be threats to your work device and could result in a massive Cyber Security issue for your organization’s Intellectual Property, and sensitive data such as Personally Identifiable Information (Nikolay Pankov, 2021).

To mitigate the issues, let’s evaluate some Microsoft Security products in this discussion.

1. Antivirus and Antispyware, or more?
A well tested antivirus and an antispyware solution at the very basic level is a must to provide basic end device Security protection. Microsoft Windows Defender can be a very good candidate solution. Windows Defender has evolved so much since its introduction. Microsoft Windows Defender was developed around the core of an older product called GIANT Antispyware. Microsoft acquired GIANT on December 2004 (Jonathan Hassell, Tony Campbell, 2007). The free version of Windows Defender provides Antivirus and Antispyware protection while the Enterprise version provides additional protections. Firewall, Email protection, Web and URL filtering, Application scan and advanced Data Telemetry and User Behavior Analytics are the examples. With all these capabilities, Windows Defender can equip a work computer with defense against network threats at the device level.

2. VPN
A Point to Site VPN connecting the work PC directly through a secure and encrypted tunnel to the company’s network can add more security to the data travelling in and out of a work computer. Microsoft Azure VPN can be an ultimate solution in this case. Microsoft Azure VPN can be configured to connect the end device to the company network as a Point to Site connection securing the data in transit through a secure and encrypted private tunnel.

3. User Behavior Analytics
User Behavior Analytics enables IT and Security teams to get alerted if a suspicious behavior is detected such as massive uploads or downloads of data, a sudden change in user’s public IP address location or impossible travel distance, and risky sign-ins using at risk devices or application could be the examples. Microsoft Defender is capable of providing the telemetry using its sensors at the end device level to a very robust service called Microsoft Cloud App Security (MCAS) service.

4. Device Compliance Policy
Security policies such as a signed-in session lock, data encryption, antivirus protection, and data travel through secure and encrypted network and such are all useless if there is not a compliance policy enforcement in place.

The Device Compliance Policy should clearly enforce all security configurations and clearly mention that any device should fail any security requirement marked as noncompliant immediately. A noncompliant device should be treated completely differently with very limited or no access to company’s IP.

Let’s now discuss some of the challenges we could encounter along the way:

1. Cost
While a simple end device protection which provides Antivirus and Antispyware protection is free but adding additional features can cost from USD $5.5 per user per month with minimum 50 license purchase to almost USD $50 per user per month including the VPN security, data telemetry and all. This could be an enormous additional cost to businesses which could potentially back them up from utilizing the product (MJ Caparas, 2021).

2. Complexity and understanding the science
Every solution comes with its pre-requisites and configuration complexities. Microsoft Security products has its own science, terms and phrases to understand.

Any incorrect and misconfiguration could result in either completely locking the company network for everyone or fully opening the network to all Cyber Threats. So proper understanding or knowing the science behind the terms and having a clear picture of the threats such as a Threat Model would help in proper configuration (Computer and Security, 2012).

3. Automation vs Human eyes
With today’s technological advances, there are capabilities to automate tasks and remedies for issues but human resource is almost equally important. A knowledgeable Cyber Security team is as important as Cyber Security tools.
