INITIAL POST

The Open Web Application Security Project (OWASP) is a non-profit organisation that diligently works to significantly improve the security of software. OWASP has been around for almost two decades and is primarily used by developers to produce more secure code (OWASP 2021).

OWASP updates their Top 10 security risks and vulnerabilities annually, you will see in Figure 1 the change of the risk landscape from 2017 to 2021. This discussion will focus on a new category for 2021 A04:2021-Insecure Design.

Figure 1. OWASP Top 10 Side by Side comparison 2017 to 2021 (OWASP 2021)

Insecure Design relates to risks in the design and architectural flaws of software. Weaknesses occur when software is developed with no risk profiling and no security controls were put in place to defend against specific attacks. Secure Design is a methodology that continuously evaluates threats and ensures code is tested and validated to prevent known attack vectors (OWASP top 10, 2021).

Out of the 40 Common Weakness Enumerations (CWEs) the below Figure 2 UML diagram depicts CWE-209 (Generation of Error Message Containing Sensitive Information) a SQL injection attack which gets successfully blocked however the error message gives away sensitive information which can be used for further attacks (OWASP 2021).

To mitigate against this issue and similar issues OWASP recommends, defining of custom/default errors that return sanitized error messages, assume all users are hostile to develop an error handling mechanism and develop specific policies on how to handle errors been logged or displayed (OWASP 2021).
